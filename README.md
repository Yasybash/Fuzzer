# Fuzzer 

  
Исследование уязвимостей ПО без исходного кода с использованием фаззинга

---

## Описание проекта

Проект представляет собой инструмент для мутационного фаззинга конфигурационного файла.

Основная цель — выявление уязвимостей (buffer overflow, integer overflow и др.) путем автоматической и ручной модификации входных данных.

---

## Функциональность фаззера

Реализованная программа выполняет:

- ручное изменение одного байта конфигурационного файла
- изменение нескольких байтов подряд
- добавление байтов в конец файла
- модификацию заголовка файла
- восстановление оригинального файла
- запуск целевого исполняемого файла под отладчиком
- автоматический режим фаззинга (mutational fuzzing)
- логирование всех изменений

---

## Автоматический режим фаззинга

Алгоритм работы:

1. Заполнение конца файла значением `0xFF`
2. Последовательное изменение байтов заголовка:
   - `0x00`
   - `0xFF`
   - `0x7F`
3. Увеличение количества изменяемых байтов
4. Перезапуск целевого приложения
5. Остановка при обнаружении ошибки

---

## Работа с конфигурационным файлом

Формат файла:

- 0x00 — номер варианта
- 0x04–0x08 — `shell_len`
- 0x08–0x0B — `dst_len`
- данные после маркера `/start` — основной буфер

Фаззер выполняет мутацию этих полей для поиска граничных состояний.


---

## Используемые технологии

- C / C++
- Windows API (CreateProcess, Debug API)
- WinDbg-style exception handling
- file mutation fuzzing
- IDA Pro + IDAPython
- DynamoRIO (code coverage)
